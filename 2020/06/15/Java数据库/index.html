JAVA的数据库编程类

JAVA SQL操作类库：

  • java.sql.*
  • javax.sql.* 这两个包只是接口类，根据数据库版本和JDBC合理选择
  • 连接字符串样例：
    • -jdbc:oracle:thin:@127.0.0.1:1521:dbname
    • -jdbc:mysql://localhost:3306/mydb
    • -jdbc:sqlserver://localhost:1433;DatabaseName=dsname

JAVA连接数据库操作步骤

  • 构建连接（搭桥）

    • 注册驱动，寻找材质，Class.forName(“········”);
    • 确定对岸目标，建桥 Connection

  • 执行操作（派个人过桥，提着篮子，去拿数据）

    • Statement 执行者
      • 使用executeQuery()执行select语句，返回结果放在ResultSet中
      • 使用executeUpdate（）执行insert/update/delete操作，返回一个int ，为被修改的行数
      • 一个Statement对象一次只能执行一个命令
    • ResultSet 结果集
      • next()判断是都还有下一条记录
      • getInt/getString/getDouble/……..
        • 可以按照索引位置，也可以按照列名

  • 释放连接（拆桥）

    • connection.close（）;

注意事项：

  • ResultSet 不能多个做笛卡尔积连接

    即最好不要嵌套使用ResultSet

  • ResultSet最好不要过百条，极其影响性能

  • ResulSet也不是一口气加载所有的select结果

  • Connection ==很昂贵==，需要及时close

  • Connection 所用的jar包和数据库要匹配

JAVA数据库编程进阶

  1. 事务处理
    • 数据库事务，Database Transaction
    • 作为单个逻辑工作单元执行的一系列操作，==要么完全的执行，要么完全的不执行==。
    • 事务必须满足ACID（==原子性，一致性，隔离性，持久性==）属性
    • 事务时数据库运行中的逻辑工作单位，由DBMS中的事务管理子系统负责事物的处理
    • JDBC事务
      • 关闭自动提交。实现多语句同一事务
      • Connection.setAutoCommit(false);
      • Connection.commot();提交事务
      • Connection.rellback();回滚事务
      • 保存点机制
        • connection.setSavepoint()
        • connection.rollback(Savepoint)

用字符串拼接方法来简化操作

PreparedStatement-安全的字符串方法

在以上代码中直接用到字符串拼接的方法来控制sql语句，这是非常危险的，如果语句中由恶意代码则会造成十分严重的后果，所以我们要引入一种更安全的方法：即使用 preparedStatement 类

String sql ="insert into book(bookid,bookname,price) values(?,?,?)";

PreparedStatement pstmt =conn.prepareStatement(sql);

pstmt.setInt(1,bookid);

pstmt.setString(2,bookname);

pstmt.setInt(3,price);

用问号代替变量，然后再分别对问号赋值，可以使得恶意代码仅仅-只变为一串字符串，而不会真正影响数据库。
