JAVA 多线程并发编程

线程的创建

  • 继承Thread类（重点）

    不建议使用，JAVA有单继承的局限性

    • 调用run方法不是多线程，调用start方法才是执行多线程并发

    • 多线程实现下载图片

  • 实现Runnable接口（重点）

    建议使用，避免JAVA单继承的局限性，灵活方便

    静态代理

  • 实现Callable接口（了解）

    • 好处：可以有返回值，可以抛出异常
    • 流程：
    1. 实现Callable接口，需要返回值类型
    2. 重写Call方法，需要抛出异常
    3. 创建目标对象
    4. 创建执行服务：ExecutorService ser = Executors.newFixedThreadPool(1);
    5. 获得结果： boolean r1 =result.get();
    6. 关闭服务: ser.shutdownNow();

    静态代理模式

Lamda表达式

  • 避免匿名内部类定义过多，去掉了一堆没有意义的代码，留下核心的逻辑
  • 其实质属于函数式编程的概念
    • （params) -> expression [表达式]
    • （params) -> expression [语句]
    • （params) -> { statements}

函数式接口

  • 定义：任何接口，如果只包含唯一一个抽象方法，那么它就是一个函数式接口

  • 对于函数式接口，我们可以用Lamda表达式来创建该接口的对象
